A contributor should understand the project quickly: what the foundation is building, where the software lives, what is ready, and where help is needed next.

## A better first impression

That is why the site includes a clear community page, a publications stream, and a visible GitHub entry point.

The site should reduce explanation overhead before the first call or the first pull request.
